## ペネトレーションテスト実践入門：OWASP Top 10で学ぶ攻撃と防御の核心  
**情報処理安全確保支援士試験対策にも役立つ！現場で使えるセキュリティ思考を養う**  

## はじめに（約200文字）  
ペネトレーションテスト（侵入テスト）は、システムの脆弱性を攻撃者視点で発見する必須の手法です。OWASP Top 10はWebアプリケーションの重大脆弱性トップ10をまとめた国際標準リスト。これらを理解しないと、実際の攻撃（例：データ漏洩やサービス停止）を防げず、情報処理安全確保支援士試験でも失点リスクがあります。本記事では「なぜ脆弱性が生まれるか」を背景から解説し、現場と試験の両方に役立つ知識を提供します。  

## 前提知識（約500文字）  
### ペネトレーションテスト（Penetration Test）とは？  
外部の攻撃者を模倣し、システムへの侵入を試みる診断手法です。脆弱性スキャン（自動診断）との違いは「実際の攻撃を再現する」点にあります。  

### OWASP（Open Web Application Security Project）とは？  
Webアプリケーションのセキュリティを推進する非営利団体です。OWASP Top 10は3年ごとに更新され（最新は2021年版）、深刻な脆弱性をランキング形式で公開します。  

### 関連用語  
- **HTTP/HTTPS**：Web通信プロトコル（RFC 7230, RFC 8446）。HTTPSは暗号化された通信です。  
- **クライアント-サーバー構造**：ユーザー（クライアント）がサーバーにリクエストを送信する基本モデル。  

```  
[ASCII図：クライアント-サーバー通信]  
  +----------+      HTTP Request      +----------+  
  |          | ---------------------> |          |  
  | Client   |                        | Server   |  
  | (Browser)| <--------------------- | (Web App)|  
  +----------+      HTTP Response     +----------+  
```  
⚠️ **試験ポイント**：ペネトレーションテストは「機密性・完全性・可用性」の評価が目的（NIST SP 800-115準拠）。  

## 基本概念（約600文字）  
### ペネトレーションテストの手法分類  
1. **ブラックボックステスト**：外部攻撃者視点。システム内部知識なしで実施。  
2. **ホワイトボックステスト**：内部者視点。設計書やソースコードを参照。  
3. **グレーボックステスト**：一部情報を保有した状態で実施。最も現実的。  

### OWASP Top 10の核心  
「開発者が犯しやすい脆弱性」を重点的に掲載。例：  
- **A01:2021-Broken Access Control**（アクセス制御の不備）  
- **A02:2021-Cryptographic Failures**（暗号化の失敗）  

```  
[ASCII図：ペネトレーションテストのフロー]  
 計画 → 情報収集 → 脆弱性診断 → 攻撃実行 → レポート作成  
          ↑                        ↑  
       (ツール利用例: Nmap)     (手動攻撃例: SQLインジェクション)  
```  
**たとえ話**：ペネトレーションテストは「家の防犯診断」に似ています。窓の鍵（アクセス制御）や金庫の強度（暗号化）を実際に試して弱点を見つける作業です。  

## 技術的な深堀り（約700文字）  
### ペネトレーションテストの具体的フロー  
1. **情報収集フェーズ**：  
   - ツール：`Nmap`（ポートスキャン, RFC 793のTCPを利用）。  
   - 例：`nmap -sV 192.168.1.1`でOS/サービスを特定。  

2. **脆弱性診断フェーズ**：  
   - OWASP ZAP（Zed Attack Proxy）で自動スキャン（CWE-89対応）。  

3. **攻撃実行フェーズ**：  
   - **SQLインジェクション**（A03:2021-Injection）の例：  
     `SELECT * FROM users WHERE id = '1' OR '1'='1';`  
     → 全ユーザーデータを不正取得（RFC 8259のJSONデータ漏洩）。  

```  
[ASCII図：SQLインジェクションのシーケンス]  
  Attacker       Web Server       Database  
     | --悪意あるクエリ--> |                |  
     |               | --クエリ実行--> |  
     | <--全データ-- | <--結果------- |  
```  
**規格参照**：  
- NIST SP 800-115（テスティング手法）  
- OWASP Testing Guide v4（実践的ガイド）  
⚠️ **試験ポイント**：インジェクション攻撃はCWE-89として登録され、支援士試験で頻出。  

### OWASP Top 10の技術的背景  
- **A05:2021-Security Misconfiguration**（設定ミス）：  
  デフォルトパスワード使用（例：admin/admin）が典型。NIST SP 800-63Bでパスワード強度が規定。  

## 攻撃・脆弱性の観点（約500文字）  
### OWASP脆弱性の悪用例  
- **Broken Access Control（A01）の攻撃フロー**：  
  1. 一般ユーザーでログイン。  
  2. URLの`user_id=123`を`user_id=admin`に変更（IDOR脆弱性）。  
  3. 管理者権限でデータ閲覧。  

```  
[ASCII図：IDOR攻撃の流れ]  
  Attacker: 一般アカウントでアクセス  
      ↓  
  管理者ページのURLを推測 (例: /admin?user_id=admin)  
      ↓  
  権限昇格 → 機密データ窃取  
```  
**有名事例**：  
- **CVE-2017-5638**：Apache Strutsの脆弱性（A06:2021-Vulnerable Components）。Equifaxのデータ漏洩（1.4億人影響）を引き起こした。  

### 試験で問われる脆弱性  
- **Cross-Site Scripting (XSS)**（A03:2021）：悪意あるJavaScriptを注入（例：`<script>alert('攻撃')</script>`）。CWE-79として登録。  

## 対策・ベストプラクティス（約500文字）  
### OWASP Top 10への対策  
1. **アクセス制御の強化**：  
   - ロールベースアクセス制御（RBAC, Role-Based Access Control）を実装。  
   - 例：管理者ページには`isAdmin=true`チェックを追加。  

2. **インジェクション対策**：  
   - プリペアドステートメントを使用（例：Javaの`PreparedStatement`）。  

```  
[ASCII図：多層防御の概念]  
  Web層: WAF導入 → アプリ層: 入力値検証 → DB層: パラメータ化クエリ  
```  
### 設定例  
- **HTTPS強制**：Apache設定で`Redirect permanent / https://...`（RFC 8446準拠）。  
- **セキュリティヘッダー**：  
  `Content-Security-Policy: default-src 'self'`でXSSを軽減。  

**ベストプラクティス**：  
- ペネトレーションテストは年1回以上実施（PCI DSS要件）。  
- OWASP Cheat Sheetを参照したコーディング。  

## 📝 試験対策ポイント（約300文字）  
下表は支援士試験の頻出テーマ。**「なぜ？」を意識して理解**しよう。  

| カテゴリ         | 頻出ポイント                     | 間違えやすい点                  |  
|------------------|----------------------------------|---------------------------------|  
| OWASP Top 10     | A01（アクセス制御）とA03（インジェクション） | 「インジェクション＝SQLのみ」と誤解（XSS含む） |  
| テスト手法       | ブラックボックスvsホワイトボックス | 「グレーボックスは内部情報なし」と誤答 |  
| ツール           | Nmap（ポートスキャン）のオプション | `-sS`（ステルススキャン）の用途を混同 |  
| 脆弱性事例       | CVE番号とOWASPカテゴリの紐付け   | CVE-2017-5638＝A06（コンポーネント脆弱性） |  

⚠️ **引っかけ問題パターン**：  
「ペネトレーションテストで発見した脆弱性は、すべて即時修正すべき」→×（リスク評価で優先度付けが必要）。  

## まとめ（約200文字）  
ペネトレーションテストは攻撃者視点で脆弱性を発見し、OWASP Top 10は重点対策すべき脆弱性を明示します。支援士試験では「A01アクセス制御」「A03インジェクション」が頻出。技術的背景（RFC/NIST）を理解し、多層防御を実践しましょう。次は「実際のツール操作（Burp Suite）」を学ぶと、実務力がさらに向上します。試験対策では「なぜ脆弱性が生まれるか」を常に問い、セキュリティエンジニアとしての基盤を固めてください！
