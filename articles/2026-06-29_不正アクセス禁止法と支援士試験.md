## 不正アクセス禁止法を技術視点で完全攻略！支援士試験の頻出ポイント付き

## はじめに
「不正アクセス禁止法」を知らずにシステム開発・運用すると、**法的リスクとセキュリティインシデントのダブルパンチ**を喰らいます！技術者であっても、この法律の「技術的要件」を理解していないと：  
- 意図せず違反行為を実装する恐れ  
- 攻撃を受けた際の法的対応を誤る  
- 試験で確実に失点する  
特に「**認証機能（Authentication）の迂回**」に関する規定は技術的紐付けが必須です。本記事では技術基盤から試験頻出ポイントまでを体系化します。

## 前提知識
### ▼ 不正アクセス禁止法とは？
**正式名称**：「不正アクセス行為の禁止等に関する法律」  
**目的**：コンピュータネットワークへの**不正なアクセス行為**を規制し、サイバー犯罪を防止する（1999年成立、2000年施行）。

### ▼ 超重要キーワード３点セット
```plaintext
+----------------+-----------------------------+
|   用語         | 定義                        |
+----------------+-----------------------------+
| 識別符号       | ユーザーを特定する情報      |
| (ID, Password) | (例：ID/PW、生体データ)      |
+----------------+-----------------------------+
| アクセス管理者 | システムの利用管理権限者    |
| (Access Admin) | (例：サーバ管理者, 企業)     |
+----------------+-----------------------------+
| アクセス制御機能| 権限に基づく利用制限機能    |
| (Access Control)| (例：RBAC, ACL)             |
+----------------+-----------------------------+
```

**⚠️ 試験必須ポイント**  
「不正アクセス＝**識別符号の不正利用＋アクセス制御機能の突破**」という構造を理解せよ！

## 基本概念
### ▼ 法律が規制する「不正アクセス行為」の技術的定義
```plaintext
[行為者]                            [対象システム]
    |                                      |
    | 1. 他人の識別符号を入力             | アクセス制御機能
    |    (例: 盗んだパスワード使用)        | (Authentication Gate)
    |----------->[認証プロセス]------------>× 拒否
    |                | 不正成功            |
    |                v                    |
    |--------->[権限範囲内リソース]        |
    |                | アクセス許可        |
    |                v                    |
    +------------->[不正アクセス成立]      |
```

**核心**：  
「**アクセス制御機能**」を**技術的に回避**することが違法性の要件。単なるパスワード漏洩自体は対象外（ただし他の法律で処罰あり）。

### ▼ たとえ話で理解
「カギ（識別符号）を盗んでドア（アクセス制御機能）を開ける行為」が違法。  
→ ドアを壊す（脆弱性攻撃）のも違法  
→ カギをコピーする行為も「**不正取得**」として禁止

## 技術的な深堀り
### ▼ アクセス制御機能の技術実装（RFC/NIST準拠）
```plaintext
           [ユーザー]
               |
               | 1. ID/PW入力
               v
[クライアント]---> [AAAサーバー] (例: RADIUS)
               |       | 認証プロトコル
               |       | (RFC 2865, RFC 2866)
               |       v
               |    [認証DB] 
               |       | LDAP (RFC 4510)
               |       | SAML (OASIS標準)
               v       v
            [アクセス許可/拒否] 
```

**技術規格のポイント**：
- **NIST SP 800-63B**：デジタルアイデンティティガイドライン（多要素認証の要件）
- **RFC 4949**：セキュリティ用語集（「認証」の厳密な定義）
- **FIDO2/WebAuthn**：パスワードレス認証の標準（W3C勧告）

### ▼ 認証プロセスの深堀り（RFC 2865 RADIUS認証フロー）
```plaintext
[ユーザー] -> [NAS] - Access-Request -> [RADIUSサーバー]
                   <- Access-Challenge  (多要素要求)
                   -- Access-Request(OTP) ->
                   <- Access-Accept/Reject
```

**⚠️ 試験頻出**：  
`Access-Challenge`は**多要素認証（MFA）** の根拠となる。不正アクセス防止にMFAが有効な理由は法的・技術的両面から出題される！

## 攻撃・脆弱性の観点
### ▼ 不正アクセスに至る攻撃フロー（例：パスワードリスト攻撃）
```plaintext
[攻撃者] -> [漏洩DB]からID/PWリスト取得
           | (例: Breached Credentials)
           v
        [ボット] -> 自動ログイン試行
           | (Credential Stuffing)
           v
[標的システム]-- 認証成功 -> [不正アクセス]
           | (アクセス制御突破)
           v
      [データ窃取] 例: CVE-2020-3452 
                   (Cisco ASA/FTD の脆弱性)
```

**有名事例**:
- **CVE-2021-44228** (Log4Shell): 認証回避型RCE
- **CVE-2020-1472** (Zerologon): ドメインコントローラ認証突破
- **JALマイレージ不正アクセス** (2023年): リスト型攻撃の典型例

## 対策・ベストプラクティス
### ▼ 技術的・法的両面からの対策マトリクス
```plaintext
+---------------------+------------------------------+-----------------------+
|  対策カテゴリ       | 技術的実装例                 | 法的対応              |
+---------------------+------------------------------+-----------------------+
| 認証強化            | ・MFAの強制 (NIST SP 800-63B)| ・アクセス管理者の    |
|                     | ・パスワードポリシー         |   義務 (法第5条)      |
+---------------------+------------------------------+-----------------------+
| 監視・検知          | ・SIEM連携 (例: Splunk)      | ・ログ保存義務        |
|                     | ・異常ログイン検知           |   (法第11条)          |
+---------------------+------------------------------+-----------------------+
| インシデント対応    | ・IPブロック (ACL設定)       | ・警察への届出        |
|                     | ・セッション切断             |   (法第12条)          |
+---------------------+------------------------------+-----------------------+
```

**実装時の必須チェックリスト**：
- [ ] 認証失敗ログにソースIPを記録（RFC 4766 syslog準拠）
- [ ] アカウントロックアウト閾値設定（例：5回失敗で30分ロック）
- [ ] 管理画面へのアクセス元制限（IP制限/VPN必須化）

## 📝 試験対策ポイント
### ▼ 絶対におさえる頻出論点表
| 論点 | 出題パターン | 引っかけポイント |
|------|--------------|------------------|
| 不正アクセス行為の定義 | 「なりすまし」と「脆弱性攻撃」の区別 | 単なるポートスキャンは対象外 |
| 識別符号の範囲 | 生体認証データ・ワンタイムパスワードの扱い | 「記憶/所持/生体」の分類問題 |
| アクセス管理者の義務 | ログ保存期間（法第11条） | 「3ヶ月」と覚えるな！(システム種別で異なる) |
| 罰則対象 | 不正取得・助長行為の具体例 | パスワードリストの販売は「助長行為」 |

**⚠️ 超頻出引っかけ**：  
「**他人のIDでログイン＝即違反**」は誤り！  
→ アクセス制御機能が**有効でないシステム**へのアクセスは対象外（法第3条ただし書き）

## まとめ
不正アクセス禁止法は「**技術的防御の法的バックボーン**」です。技術者として押さえる核心は：
1. 違反の要件＝**アクセス制御機能の突破技術**
2. 認証システムは **RFC/NIST 準拠**がリスク軽減に直結
3. 試験では **「なりすまし」「脆弱性悪用」「助長行為」の区別**が頻出
