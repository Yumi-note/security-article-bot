# ファイアウォールの種類：パケットフィルタリングからNGFWまで

## はじめに

ファイアウォールは、ネットワークセキュリティを確保するための重要なコンポーネントです。近年、サイバー攻撃の増加とともに、ファイアウォールの重要性も高まっています。しかし、ファイアウォールにはパケットフィルタリングやアプリケーションゲートウェイなど種類があります。違いの理解が不足している場合は、セキュリティ強化に効果的な対策が取れない可能性があります。

本稿では、ファイアウォールの基本概念から、具体的な種類や技術的な詳細について解説し、さらに攻撃の観点や対策についても詳しく説明します。

## 前提知識

ファイアウォールを理解する前に、まずはネットワークの基本的な構成と関連する用語を理解する必要があります。

ネットワークは、ホスト（サーバー、クライアント）やルーター、スイッチなどの機器を接続して構成されています。ホスト間のデータ転送は、OSI参照モデルのプロトコルスタックに基づいて実現されます。

ここで重要な用語をいくつか説明します。

* **パケット**：データを転送する際に細分化されたデータの単位
* **プロトコル**：データ転送の規則。TCP/IPプロトコルスタックが最も一般的
* **ポート番号**：プロセスを識別するための番号。ウェブサービスには80番ポートが一般的
* **IPアドレス**：ホストを識別するためのアドレス。IPv4（32bit）とIPv6（128bit）がある

## 基本概念

ファイアウォールは、ネットワークの境界に設置され、不正アクセスを防止するための機器またはソフトウェアです。ファイアウォールには、以下の基本的な種類があります。

* **パケットフィルタリングファイアウォール**：パケットのヘッダ情報（送信元IP、宛先IP、ポート番号など）を基にフィルタリングする
* **アプリケーションゲートウェイファイアウォール**：アプリケーション層のプロトコル（HTTP、FTPなど）を解析し、フィルタリングする
* **ステートフルファイアウォール**：パケットの送信元と宛先の関係性を把握し、フィルタリングする

以下は、パケットフィルタリングファイアウォールの基本的なフローを表すASCII図です。
```

  +---------------+
           |
           |
           v
  +---------------+
  |  ヘッダ情報チェック  |
  |  (送信元IP、宛先IP、  |
  |   ポート番号など)     |
  +---------------+
           |
           |
           v
  +---------------+
  |  ルールマッチング  |
  |  (許可、拒否、あるいは  |
  |   ログ出力など)       |
  +---------------+
           |
           |
           v
  +---------------+
  |  パケット通過 or 拒否  |
  +---------------+
```
## 技術的な深堀り

パケットフィルタリングファイアウォールの具体的な動作は、RFC 791（Internet Protocol）やRFC 793（Transmission Control Protocol）に基づいて実現されています。

以下は、TCPヘッダの構造を表すASCII図です。

## 攻撃・脆弱性の観点

ファイアウォールは、不正アクセスを防止するための重要なコンポーネントですが、以下のような攻撃にも脆弱です。

* **パケットフィルタリングの迂回攻撃**：パケットのヘッダ情報を改ざんすることで、ファイアウォールを迂回する攻撃
* **バッファーオーバーフロー攻撃**：ファイアウォールのバッファに不正なデータを詰め込み、クラッシュさせる攻撃

以下は、パケットフィルタリングの迂回攻撃のシーケンス図を表すASCII図です。

  +---------------+
           |

  |  ファイアウォール  |
  +---------------+
           |
           |

## 対策・ベストプラクティス

ファイアウォールを強化するための対策として、以下のようなベストプラクティスがあります。
