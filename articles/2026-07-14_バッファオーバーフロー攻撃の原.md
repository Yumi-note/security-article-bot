# バッファオーバーフロー攻撃の原理とメモリ保護機構

## はじめに

バッファオーバーフロー攻撃は、コンピュータセキュリティの分野では古典的な脆弱性の一つです。この攻撃が成功すると、攻撃者は任意のコードを実行することができ、システムを完全に乗っ取る可能性があります。情報処理安全確保支援士の試験では、バッファオーバーフロー攻撃に関する知識が問われることが多いです。この記事では、バッファオーバーフロー攻撃の原理とメモリ保護機構について解説します。

## 前提知識

### バッファとメモリ

コンピュータでは、プログラムが実行される際にメモリが使用されます。メモリとは、データを一時的に保持するための領域です。プログラムは、メモリ上のバッファと呼ばれる領域にデータを格納します。

```
          +---------------+
          |  メモリ   |
          +---------------+
                  |
                  |
                  v
+---------------+---------------+
|  バッファ  |  その他のデータ  |
+---------------+---------------+
```

### スタックとヒープ

メモリには、スタックとヒープの2つの領域があります。スタックは、関数の呼び出しや変数の格納に使用されます。ヒープは、動的にメモリを割り当てるために使用されます。

```
          +---------------+
          |  メモリ   |
          +---------------+
                  |
                  |
                  v
+---------------+---------------+
|  スタック  |  ヒープ  |
+---------------+---------------+
```

## 基本概念

### バッファオーバーフロー

バッファオーバーフローは、バッファの容量を超えてデータを格納した場合に発生します。 ATTACKERは、バッファの容量を超えてデータを送信し、バッファをオーバーフローさせます。すると、攻撃者のデータがバッファの外側に格納され、システムを乗っ取ることができます。

```
          +---------------+
          |  バッファ  |
          +---------------+
                  |
                  |
                  v
+---------------+---------------+
|  正しいデータ  |  攻撃者のデータ  |
+---------------+---------------+
                  ^
                  |
                  |
                  v
          +---------------+
          |  システムを乗っ取る  |
          +---------------+
```

### メモリ保護機構

メモリ保護機構は、バッファオーバーフローを防止するために使用されます。以下に、メモリ保護機構の種類を示します。

*   アドレス空間分離（Address Space Layout Randomization: ASLR）
*   データ実行防止（Data Execution Prevention: DEP）
*   スタックガード

これらの機構は、バッファオーバーフローを防止し、システムのセキュリティを強化します。

## 技術的な深堀り

### バッファオーバーフロー攻撃のフロー

バッファオーバーフロー攻撃は、以下のステップで行われます。

1.  攻撃者は、バッファオーバーフローを起こすためにデータを送信します。
2.  バッファは、データを格納するために割り当てられます。
3.  攻撃者のデータは、バッファの容量を超えて格納されます。
4.  バッファの外側に格納されたデータは、システムを乗っ取るために使用されます。

```
          +---------------+
          |  攻撃者  |
          +---------------+
                  |
                  |
                  v
+---------------+---------------+
|  データの送信  |  バッファの割り当て  |
+---------------+---------------+
                  |
                  |
                  v
          +---------------+
          |  バッファのオーバーフロー  |
          +---------------+
                  |
                  |
                  v
          +---------------+
          |  システムの乗っ取り  |
          +---------------+
```

### メモリ保護機構の詳細

メモリ保護機構は、以下の方法でバッファオーバーフローを防止します。

*   ASLR: プロセスのアドレス空間をランダム化することで、攻撃者が特定のアドレスにデータを格納することを困難にします。
*   DEP: データを実行できない領域に格納することで、攻撃者がコードを実行することを防止します。
*   スタックガード: スタックの末尾にキャッシュした値を格納することで、攻撃者がスタックをオーバーフローさせることを防止します。

これらの機構は、バッファオーバーフローを防止し、システムのセキュリティを強化します。

## 攻撃・脆弱性の観点

### バッファオーバーフロー攻撃の例

バッファオーバーフロー攻撃は、以下の例のように行われます。

*   Heartbleed: OpenSSLのバッファオーバーフローにより、暗号化されたデータが漏洩しました。
*   Shellshock: Bashのバッファオーバーフローにより、攻撃者が任意のコードを実行できました。

これらの攻撃は、バッファオーバーフローを起こすために使用されました。

### バッファオーバーフロー攻撃の対策

バッファオーバーフロー攻撃を防止するには、以下の対策を講じます。

*   バッファのサイズを適切に設定する
*   データの入力チェックを実施する
*   メモリ保護機構を使用する

これらの対策は、バッファオーバーフローを防止し、システムのセキュリティを強化します。
