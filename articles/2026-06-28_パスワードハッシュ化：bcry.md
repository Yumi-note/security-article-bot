## パスワードハッシュ化の最前線：bcrypt・scrypt・Argon2を徹底比較！～支援士試験対策付き～

## はじめに（200文字）
パスワード管理は認証システムの要です。平文パスワードを保存すると、データベース漏洩時に**全アカウントが即時乗っ取られる**リスクがあります。実際、2020年の**Twitter社内部犯行事件**では平文パスワード悪用で著名人アカウントがハッキングされました。適切なハッシュ化が行われていれば、攻撃者は解読に何年もかかり被害を最小化できます。本記事では現代の主要ハッシュアルゴリズムを比較し、設計・実装のベストプラクティスを解説します。

## 前提知識（500文字）
### ハッシュ関数の基本性質
- **一方向性関数（One-way function）**：出力から入力を逆算不可能
- **耐衝突性（Collision resistance）**：異なる入力で同じ出力を生成困難
- **ソルト（Salt）**：同じパスワードでも異なるハッシュを生成するための乱数  
  `パスワード + ソルト → ハッシュ関数 → ハッシュ値`

### ストレッチング（Stretching）
ブルートフォース攻撃対策に計算を意図的に重くする仕組み。ハッシュ関数を複数回繰り返すことで処理時間を増加させます。

```
入力パスワード
   │
   ▼
[ソルト生成]──┐
   │          ▼
   │       [ハッシュ関数]←─[反復処理]←─┐
   │          │ (n回繰り返し)        │
   ▼          ▼                       │
[比較] ←─ 保存済みハッシュ値          │
              ▲                       │
              └───────────────────────┘
```

## 基本概念（600文字）
### パスワードハッシュの要件（NIST SP 800-63B準拠）
1. **ソルト付与**：長さ32ビット以上、ユーザー毎に固有
2. **ストレッチング**：反復処理で計算コスト向上
3. **メモリ制約**：並列処理を困難化（GPU/ASIC攻撃対策）

### 主要アルゴリズムの特徴比較
| 方式   | 開発年 | 主な特徴                     |
|--------|--------|------------------------------|
| bcrypt | 1999   | ブロック暗号ベース、CPU負荷重視 |
| scrypt | 2009   | メモリ負荷を導入、GPU耐性    |
| Argon2 | 2015   | メモリ/スレッド多重防御、国際公募採用 |

```
[入力]  
   │  
   ├─[ソルト結合]  
   │  
   ▼  
[コア処理]←─[計算コストパラメータ]  
   │  
   ▼  
[出力ハッシュ]  
```

⚠️ **試験頻出**：Argon2は**Password Hashing Competition（PHC）** で優勝し、現在のデファクトスタンダード

## 技術的な深堀り（700文字）
### bcryptの構造（Blowfish暗号ベース）
``` 
キー拡張
  │
  ▼
[EksBlowfish]─┬─[ソルト]  
  │           └─[コスト設定]（反復回数=2^n）
  ▼
ハッシュ値（60文字）
```
- **コストファクター**：4～31の整数（デフォルト=10→2^10=1024回反復）
- RFC: 非公式（元論文"Future-Adaptable Password Scheme"）

### scryptのメモリハード設計
``` 
入力 + ソルト
   │
   ▼
[PBKDF2] → [ROMix関数] → 最終ハッシュ
      ▲         │
      └─[大量メモリ使用]  
```
- パラメータ：N（メモリコスト）, r（ブロックサイズ）, p（並列度）
- RFC 7914で標準化

### Argon2の多重防御
``` 
入力
   │
   ▼
[メモリ初期化]←─[並列スレッド]
   │
   ▼
[圧縮関数処理]（メモリ依存）
   │
   ▼
[最終ハッシュ生成]
```
- **タイプ選択**：Argon2d（データ依存）, Argon2i（攻撃耐性）, Argon2id（ハイブリッド）
- **パラメータ**：m（メモリサイズ）, t（反復回数）, p（並列度）
- RFC 9106で標準化

## 攻撃・脆弱性の観点（500文字）
### 主要攻撃手法
1. **レインボーテーブル攻撃**：事前計算済みハッシュ辞書による逆引き  
   → ソルトで完全防止可能
2. **GPU/ASICブルートフォース**：安価なハードウェアで並列処理  
   → メモリハード関数で対策（scrypt/Argon2）

```
攻撃者
   │
   ▼
[漏洩DB取得] → [ハッシュ値抽出] → [辞書攻撃/総当たり]  
                              ▲
                              └─[GPUクラスタ]（1秒あたり数億試行）
```

### 実被害事例
- **LinkedIn漏洩事件（2012年）**：SHA-1ハッシュ600万件が解読され平文パスワード流出（CVE-2012-5971）
- **Adobe社データ漏洩（2013年）**：3DES暗号化パスワードが逆算可能な形式で保存

⚠️ **試験対策**：bcryptは**タイミング攻撃**に弱い→Argon2/scrypt推奨（NIST SP 800-63B）

## 対策・ベストプラクティス（500文字）
### 実装ガイドライン
1. **アルゴリズム選択**：新規システムは**Argon2id**を優先（NIST推奨）
2. **パラメータ設定例**（Argon2）：
   - メモリコスト(m)：19～64MB
   - 反復回数(t)：1～3回
   - 並列度(p)：コア数に応じて2～4
3. **ソルト管理**：長さ16バイト以上、CSPRNG（Cryptographically Secure PRNG）で生成

### 言語別実装例
```python
# Python (argon2-cffiライブラリ)
from argon2 import PasswordHasher
ph = PasswordHasher(
    memory_cost=65536,  # 64MB
    time_cost=2,
    parallelism=2
)
hash = ph.hash("password123")
```
