# DNSポイズニングの仕組みとDNSSECによる防御

## はじめに

インターネットでのデータ送受信は、IPアドレスを使用して行われます。しかし、人間はIPアドレスを直接記憶して使うことは困難です。そこで、ドメイン名からIPアドレスを解決する仕組みが必要になります。この仕組みが DNS（Domain Name System）です。しかし、DNSにはセキュリティ上のリスクがあることが知られています。その一つがDNSポイズニングです。DNSポイズニングが攻撃者によって悪用されると、ユーザーが意図しないサイトへ接続されたり、감染したりする危険性があります。このため、DNSポイズニングの仕組みと、DNSSECによる防御について理解することが重要です。

## 前提知識

### DNSとは

DNS（Domain Name System）は、ドメイン名からIPアドレスを解決する仕組みです。ドメイン名は人間が使いやすい名前ですが、コンピューターはIPアドレスを使用してデータ送受信を行います。

```
ドメイン名  <- DNS ->  IPアドレス
```

### DNSの構造

DNSは階層構造になっています。ドメイン名の各部分はラベルと呼ばれます。ドメイン名の末尾はドット (`.`) で終わります。ドットの左側が親ドメイン、右側が子ドメインです。DNSサーバーはこの階層構造に基づいてドメイン名を解決します。

```
example.com.  <- ルートDNSサーバー
|           |
example     com
```

### DNSプロトコル

DNSプロトコルは、クライアントとDNSサーバー間で使用されるプロトコルです。クライアントはドメイン名を送信し、DNSサーバーはIPアドレスを返します。

```
クライアント  <- DNS Query ->  DNSサーバー
クライアント  <- DNS Response ->  DNSサーバー
```

## 基本概念

### DNSポイズニング

DNSポイズニングは、DNSサーバーに偽の情報を送信し、ユーザーが意図しないサイトへ接続されるようにする攻撃の一つです。DNSサーバーはクエリに対して正しいIPアドレスを返すことを期待していますが、攻撃者は偽のIPアドレスを送信します。

```
クライアント  <- DNS Query ->  DNSサーバー
DNSサーバー  <- 偽のIPアドレス ->  クライアント
```

### DNSSEC

DNSSEC（Domain Name System Security Extensions）は、DNSのセキュリティを強化するための仕組みです。DNSSECでは、DNSサーバーは署名付きのレスポンスを返します。クライアントは署名を検証し、正しいIPアドレスであることを確認できます。

```
クライアント  <- DNS Query ->  DNSサーバー
DNSサーバー  <- 署名付きのレスポンス ->  クライアント
クライアント  <- 検証 ->  正しいIPアドレス
```

## 技術的な深堀り

### DNSSECの仕組み

DNSSECでは、DNSサーバーは署名付きのレスポンスを返します。署名は、DNSKEYと呼ばれる秘密鍵を使用して生成されます。クライアントは署名を検証し、正しいIPアドレスであることを確認できます。

```
DNSサーバー  <- DNSKEY ->  署名生成
DNSサーバー  <- 署名付きのレスポンス ->  クライアント
クライアント  <- 検証 ->  正しいIPアドレス
```

### NSEC/NSEC3

NSEC（Next Secure）/NSEC3（Next Secure 3）は、DNSSECの拡張仕様です。NSEC/NSEC3では、DNSサーバーはゾーンファイルに未登録ドメインに関する情報を含めることができます。クライアントは、署名付きのレスポンスを受信した場合に、正しいIPアドレスであることを確認できます。

```
DNSサーバー  <- NSEC/NSEC3 ->  ゾーンファイル
クライアント  <- 署名付きのレスポンス ->  DNSサーバー
クライアント  <- 検証 ->  正しいIPアドレス
```

### RFCIntArray

RFCIntArrayは、DNSSECのドメイン名解決プロセスを定義したRFC文書の番号です。

```
RFC 4033
RFC 4034
RFC 4035
```

## 攻撃・脆弱性の観点

### Kaminsky Attack

Kaminsky Attackは、DNSポイズニング攻撃の一つです。攻撃者は、DNSサーバーに偽の情報を送信し、ユーザーが意図しないサイトへ接続されるようにします。

```
クライアント  <- DNS Query ->  DNSサーバー
DNSサーバー  <- 偽のIPアドレス ->  クライアント
```

### DNS Amplification Attack

DNS Amplification Attackは、DNSサーバーを標的としたDoS/DDoS攻撃の一つです。攻撃者は、DNSサーバーに大量のDNSクエリを送信し、DNSサーバーのリソースを枯渇させます。

```
攻撃者  <- DNS Query ->  DNSサーバー
DNSサーバー  <- レスポンス ->  攻撃者
```

## 対策・ベストプラクティス

### DNSSECの実装

DNSSECの実装は、DNSサーバーのセキュリティを強化するためには不可欠です。DNSサーバーは署名付きのレスポンスを返すことができるようにする必要があります。

### キーの管理

キーの管理は、DNSSECのセキュリティを強化するためには重要です。DNSKEYと呼ばれる秘密鍵を安全に管理する必要があります。

### ゾーンファイルの管理

ゾーンファイルの管理は、DNSサーバーのセキュリティを強化するためには重要です。ゾーンファイルに未登録ドメインに関する情報を含めることができるようにする必要があります。

## 試験対策ポイント

| カテゴリー | ポイント |
| --- | --- |
| DNSポイズニング | DNSサーバーに偽の情報を送信し、ユーザーが意図しないサイトへ接続されるようにする攻撃の一つです。 |
| DNSSEC | DNSのセキュリティを強化するための仕組みです。DNSサーバーは署名付きのレスポンスを返します。 |
| NSEC/NSEC3 | DNSSECの拡張仕様です。DNSサーバーはゾーンファイルに未登録ドメインに関する情報を含めることができます。 |

## まとめ

DNSポイズニングは、DNSサーバーに偽の情報を送信し、ユーザーが意図しないサイトへ接続されるようにする攻撃の一つです。DNSSECは、DNSのセキュリティを強化するための仕組みです。DNSサーバーは署名付きのレスポンスを返します。NSEC/NSEC3は、DNSSECの拡張仕様です。DNSサーバーはゾーンファイルに未登録ドメインに関する情報を含めることができます。DNSSECの実装、キーの管理、ゾーンファイルの管理は、DNSサーバーのセキュリティを強化するためには不可欠です。
