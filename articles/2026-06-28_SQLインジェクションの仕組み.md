# SQLインジェクションの仕組みと完全な対策

## はじめに

SQLインジェクションはWebアプリケーションの脆弱性の代表格の一つです。悪意のあるユーザーがSQL文を含む入力tingを攻撃対象システムに送信し、疎外されていないデータベースの管理者ユーザーが実行可能なコマンドを実行できるようにする可能性があります。OWASP TOP 10には2017年版では第1位にランクインしています。

この記事では、SQLインジェクションの仕組みと完全な対策について解説します。

## 基本概念

SQLインジェクションは、攻撃者がWebアプリケーションのデータベースと直接接続することにより、情報を漏洩させたり、操作したりする可能性があります。主に、インジェクション攻撃は、SQL文を含む入力通として扱われないように設計されたWebアプリケーションに存在します。

### SQL インジェクションの過程

1.  **SQL入力**:攻撃者がWebアプリケーションにSQL入力を送信する。
2.  **SQL解析**:WebアプリケーションがSQL入力を検証し、解析し、データベースに通信します。
3.  **SQL実行**:攻撃者が実行可能なSQLコマンドを挿入する。

### 攻撃と脆弱性

SQLインジェクション攻撃の代表的な方法は:

*   **診断対象**SQLインジェクション攻撃
    *   暗号化されたパスワードを解読
    *   利用者の個人データを取得
    *   DBを書き込む権限を取得

*   **ジャストインケース**SQLインジェクション攻撃
    *   転送ファイルの取得

脆弱性は、次のとおりです。

*   **未記入の入力**:未記入の入力を含むアプリケーションでデータベース操作を行う。
*   **高度な入力**:入力を含むアプリケーションでデータベース操作を行う。
*   **リダイレクト**:SQLのQUERY_STRINGのみを動的にSQL文に含む。

## 技術的な深堀り

SQLインジェクションは主に、WebアプリケーションがSQL文を含む入力を、効率の向上のために動的にSQL文に含ませることによって発生します。ここで、SQLのSQL Server関数BETWEEN、OR、NOT、USING、INを対象とします。

SQL ServerのBETWEENは`WHEREカラム BETWEEN 値1 AND 値2` を意味しますが、`WHEREカラム BETWEEN '値1' AND '値2'` は脆弱性を持つ。

### 攻撃者がインジェクションを利用する方法

攻撃者はインジェクションを使用してデータベース管理者が権限を持つSQLコマンドを実行します。次の例を使用します。

SQLインジェクションを悪用することができるのは、次の3つあります。

*   **Trojan HorseのSQLインジェクション**:ビットレベルデータベースに対するコントロール権限を得るためにSQLインジェクションを使用します。
*   **End-EnderのSQLインジェクション**:ユーザーデータベース管理者を上書きするためにSQLインジェクションを使用します。
*   **投稿型SQLインジェクション**:SQLインジェクションを使用して投稿タイプのデータベースに対する管理権限を取得します。

## 攻撃・脆弱性の観点

以下の脆弱性を持つSQLインジェクションを防ぐ必要があります。

*   SQLインジェクション対策:以下のニーズはすべてSQLインジェクション対策の重要な部分です。

    1.  **なし条件**
    2.  **なしパラメータ入力**
    3.  **なしサニタイズ**
    4.  **なし自動コミット**

## 対策・ベストプラクティス

SQLインジェクション対策のための最も効果的な方法は、入力を検証して解釈することです。特定の利用者のペアの値を検証する必要があるかどうかを確認することも必要です。

SQLインジェクションを防ぐには、以下のステップを実行します。

1.  **SQL入力チェック**:Webアプリケーションは、SQL入力を検証し、解釈し、データベースと通信します。
2.  **パラメータ入力**:Webアプリケーションは、SQL入力を含むパラメータ入力を検証します。
3.  **SQLサニタイズ**:Webアプリケーションは、SQL入力を検証し、SQLサニタイズします。
4.  **SQLコミット**:Webアプリケーションは、SQLコミットが実行されるように設計されています。

以下のベストプラクティスは、SQLインジェクションを防ぐために必須です。

1.  **始まりの条件**:Webアプリケーションは、入力を検証し、SQL入力を検証し、データベースに検証します。
2.  **ソフト入力**:Webアプリケーションは、SQL入力を含むソフト入力を検証します。
3.  **SQLサニタイズ**:Webアプリケーションは、SQL入力を検証し、SQLサニタイズします。
4.  **SQLコミット**:Webアプリケーションは、SQLコミットが実行されるように設計されています。

## 試験対策ポイント

SQLインジェクションの問題は、上記の主な特徴に触れている場合に最も頻繁に頻出します。

1.  **未記入入力**
2.  **高度な入力**
3.  **リダイレクト**

WebアプリケーションのSQLインジェクションを防ぐには、上記の根本的な条件を使用して入力を検証する必要があります。
