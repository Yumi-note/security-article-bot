# インシデントレスポンスの手順：NIST SP 800-61の実践

## はじめに
サイバー攻撃や情報漏えいのようなセキュリティインシデントが発生した場合、適切な対応が求められます。インシデントレスポンスを遅らせたり、不適切に処理すると、被害が拡大したり、再発しないようにするための貴重な機会を失ったりします。そのため、NIST SP 800-61が提供するインシデントレスポンスの手順を理解することが重要です。本稿では、インシデントレスポンスの手順について解説し、実際の現場で活用するための指針を提供します。

## 前提知識

### インシデントレスポンスの定義
インシデントレスポンスとは、セキュリティインシデントが発生した際に実施される対応策のことです。インシデントレスポンスの目的は、インシデントの被害を最小限に抑え、インシデントが再発することを防ぐことです。

### セキュリティインシデントの例
セキュリティインシデントの例として、以下のようなものがあります。

* unauthorized access (不正アクセス)
* malware infection (マルウェア感染)
* data breach (データ漏えい)
* denial of service (サービス妨害)

|  (不正アクセス)     |
+---------------+

+---------------+
|  data breach      |
|  (データ漏えい)    |
+---------------+
|  denial of service  |
|  (サービス妨害)  |
+---------------+
```

### インシデントレスポンス計画の重要性
インシデントレスポンス計画は、セキュリティインシデントが発生した際に迅速かつ効果的に対応するための枠組みを提供します。この計画には、インシデントの分類、通報の手順、対応チームの構成などが含まれます。

ASCII図: インシデントレスポンス計画の構成要素

```
+---------------+

|  (インシデントの分類) |
+---------------+
|  reporting procedure    |
|  (通報の手順)      |
+---------------+
|  incident response team |
|  (対応チームの構成) |
+---------------+
```

## 基本概念

### インシデントレスポンスの手順
NIST SP 800-61は、インシデントレスポンスの手順を以下の4つのステージに分類しています。

1. **Detection and Reporting** (検知と通報)
2. **Containment, Eradication, Recovery, and Post-Incident Activities** (包含、排除、復旧、及び事後活動)
3. ** Lessons Learned** (教訓の抽出)

|  Lessons Learned |
+---------------+
```

## 攻撃・脆弱性の観点

### インシデントレスポンスの弱点
インシデントレスポンスには以下のような弱点があります。

* 不十分な計画と準備
* 不適切な通報と対応
* 失敗した包含と排除
* 不足な事後活動
