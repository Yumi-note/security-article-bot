**IDS/IPSの仕組みとシグネチャ型・アノマリ型の違い**

## はじめに

インターネットやネットワークを介して情報システムにアクセスする際、セキュリティ対策を講じることは重要です。その一環として、不正アクセスやサイバー攻撃を検知・阻止するために使用される IDS (Intrusion Detection System) と IPS (Intrusion Prevention System) があります。しかし、IDS/IPS の仕組みや種類、特にシグネチャ型とアノマリ型の違いについて、理解が不足している場合は、セキュリティ上のリスクを抱え込むことになります。この記事では、IDS/IPS の基礎知識から始め、仕組みや違い、そして実際の攻撃事例と対策までを解説します。

## 前提知識

**IDS/IPSの基本**

IDS (Intrusion Detection System) と IPS (Intrusion Prevention System) は、ネットワークトラフィックを監視して不正アクセスやサイバー攻撃を検知するシステムです。

**パケット**

ネットワークトラフィックはパケットという単位で構成されています。パケットは、ヘッダー (送信元/送信先情報など) とペイロード (実際のデータ) で構成されています。

**プロトコル**

ネットワーク通信では、TCP (Transmission Control Protocol) や UDP (User Datagram Protocol) などのプロトコルが使用されます。これらのプロトコルは、データの送受信を管理するためのルールです。

## 基本概念

**IDS/IPSの仕組み**

IDS/IPS には、シグネチャ型 (Signature-based) とアノマリ型 (Anomaly-based) の 2 種類があります。

**シグネチャ型**

シグネチャ型 IDS/IPS では、事前に登録された攻撃パターン (シグネチャ) とトラフィックを比較して不正アクセスを検知します。

[ASCII図]

```
+---------------+
|  シグネチャ  |
+---------------+
                  |
                  |
                  v
+---------------+       +---------------+
|  トラフィック  |       |  許可/拒否  |
+---------------+       +---------------+
```

**アノマリ型**

アノマリ型 IDS/IPS では、正常なトラフィックパターンから逸脱したものを不正アクセスと判定します。

[ASCII図]

```
+---------------+
|  正常パターン  |
+---------------+
                  |
                  |
                  v
+---------------+       +---------------+
|  トラフィック  |       |  許可/拒否  |
+---------------+       +---------------+
```

## 技術的な深堀り

**パケット解析**

IDS/IPS では、パケットを解析してヘッダー情報やペイロードを抽出します。

[ASCII図]

```
+---------------+
|  パケット    |
+---------------+
                  |
                  |
                  v
+---------------+       +---------------+
|  ヘッダー    |       |  ペイロード  |
+---------------+       +---------------+
```

**ルールベース**

IDS/IPS では、ルールベースに基づいてトラフィックを判定します。ルールは、特定の条件 (送信元/送信先IPアドレス、ポート番号など) に基づいて設定されます。

## 攻撃・脆弱性の観点

**SQLインジェクション**

SQLインジェクションは、ウェブアプリケーションの入力フォームに悪意のあるSQLコードを.inject して、データベースの情報を取得したり操作したりする攻撃です。

[ASCII図]

```
+---------------+
|  ユーザー入力  |
+---------------+
                  |
                  |
                  v
+---------------+       +---------------+
|  SQLコード  |       |  データベース  |
+---------------+       +---------------+
```

**クロスサイトスクリプティング (XSS)**

XSS は、ウェブアプリケーションに悪意のあるスクリプトを仕込んで、ユーザーのブラウザで実行させる攻撃です。

[ASCII図]

```
+---------------+
|  ユーザー入力  |
+---------------+
                  |
                  |
                  v
+---------------+       +---------------+
|  スクリプト  |       |  ブラウザ    |
+---------------+       +---------------+
```

## 対策・ベストプラクティス

**IDS/IPSの設定**

IDS/IPS の設定では、シグネチャやルールを正しく設定する必要があります。

**ファイアウォール**

ファイアウォールは、ネットワーク境界で不正アクセスを阻止するためのセキュリティ機器です。

**パッチ管理**

ソフトウェアやシステムのパッチを定期的に適用することで、脆弱性を修正することが重要です。
