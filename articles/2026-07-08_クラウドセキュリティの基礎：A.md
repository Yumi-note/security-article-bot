# クラウドセキュリティの基礎：AWSの責任共有モデル

## はじめに

クラウドセキュリティは、現代の情報処理の重要な要素の一つです。AWS (Amazon Web Services) を利用する組織や個人は、クラウドセキュリティを理解する必要があります。なぜなら、セキュリティとは共有責任であり、AWS とユーザーが共同で責任を担うからです。しかし、この責任共有モデルがどう機能するのか、知らないとどのようなリスクがあるのか、理解している人は多くありません。この記事では、AWS の責任共有モデルについて、基礎から詳しく解説します。

## 前提知識

クラウドコンピューティングとは、インターネット上で提供されるコンピュータサービスの一種です。AWS は、クラウドコンピューティングの先駆的サービスの一つです。セキュリティとは、機密性 (Confidentiality)、完全性 (Integrity)、可用性 (Availability) の 3 要素からなります。 

  + 機密性：認証された機関のみが情報にアクセスできること
  + 完全性：情報が改変されないこと
  + 可用性：情報が必要な時に利用可能であること

AWS は、セキュリティを、データセンターのセキュリティ、ネットワークセキュリティ、データのセキュリティという 3 つの分野に分けており、これらのエリアそれぞれに対して詳細な設計と運用が行われています。

クラウドセキュリティサービス(Cloud Security Responsibility Matrix)の責任共有モデルによると、

**AWSの責任の範囲**

| 分野        | 内容 |
| :-------------------  | :------------------- |
| Facilities           | 物理的なデータセンター |
| Network               | ネットワークセキュリティ   |
| Compute               | サーバー、ストレージ    |
| Database              | データベース             |
| Identity & Access     | ID、認証、アクセス制御 |
| IAM Users, Groups, Roles | ユーザー、グループ、ロール   |

AWS の責任とユーザーの責任を明確にすることで、セキュリティ体制の構築がしやすくなります。

## 基本概念

AWS の責任共有モデルは、セキュリティを 3 つのレベルに分けます。

1.  **セキュリティ OF クラウド**：AWS がデータセンターのセキュリティ、ネットワークのセキュリティ、サーバーのセキュリティを担います。
2.  **セキュリティ IN クラウド**：ユーザーがデータのセキュリティ、アプリケーションのセキュリティ、IAM ユーザーのセキュリティを担います。

例えば、AWS がデータセンターのセキュリティを担う一方で、ユーザーがデータの暗号化を担います。

  + S3：オブジェクトストレージ、デフォルトで暗号化されるが、ユーザー側で管理することも可能。
  + EBS：ブロックストレージ、ユーザー側で暗号化を選択する必要がある。

ユーザーは、AWS の提供するセキュリティサービスを活用することで、セキュリティを強化できます。

  + IAM (Identity and Access Management)：認証、アクセス制御を行うサービス
  + Cognito：ユーザーの認証、許可を行うサービス

## 技術的な深堀り

AWS の責任共有モデルは、ISO/IEC 27001 という国際標準に基づいています。この標準では、情報セキュリティマネジメントシステム (ISMS) の構築が規定されています。

### IAM (Identity and Access Management)

AWS の IAM は、認証と許可を行うサービスです。IAM ユーザーを作成することで、ユーザー別にアクセス権限を管理できます。
