**XSS（クロスサイトスクリプティング）の種類と対策**

## はじめに

Webアプリケーションは、ユーザーが入力したデータを適切に処理せずに表示すると、悪意のあるスクリプトが実行される可能性があります。これはクロスサイトスクリプティング（XSS）と呼ばれ、ウェブセキュリティの重要な脆弱性の一つです。XSSの攻撃により、ユーザーの機密情報が盗まれたり、未承認の操作が行われたりする恐れがあります。情報処理安全確保支援士の試験対策を始める前に、XSSの種類と対策を理解することが重要です。

## 前提知識

**Webアプリケーションの構成**

```
          +---------------+
          |  Webブラウザ  |
          +---------------+
                  |
                  |  HTTPリクエスト
                  v
          +---------------+
          |  Webサーバー  |
          |  (Apache, Nginx)|
          +---------------+
                  |
                  |  HTTPレスポンス
                  v
          +---------------+
          |  Webアプリケーション|
          |  (PHP, Ruby, Python)|
          +---------------+
                  |
                  |  データベース
                  v
          +---------------+
          |  データベース  |
          |  (MySQL, PostgreSQL)|
          +---------------+
```

**HTTPプロトコル**

HTTP（HyperText Transfer Protocol）は、WebブラウザとWebサーバー間でデータをやりとりするためのプロトコルです。HTTPリクエストには、ユーザーが入力したデータが含まれます。

## 基本概念

**XSSの種類**

XSSには、主に3種類があります。

* **反射型XSS（Reflected XSS）**: 悪意のあるスクリプトが、Webサーバーを経由してユーザーのブラウザに返されます。
* **ストアド型XSS（Stored XSS）**: 悪意のあるスクリプトが、Webサーバーに保存されます。
* **DOM型XSS（DOM-based XSS）**: 悪意のあるスクリプトが、WebブラウザのDOM（Document Object Model）を操作します。

**XSSの仕組み**

```
          +---------------+
          |  ユーザー  |
          +---------------+
                  |
                  |  悪意のあるデータ
                  v
          +---------------+
          |  Webサーバー  |
          |  (XSS脆弱性)  |
          +---------------+
                  |
                  |  悪意のあるスクリプト
                  v
          +---------------+
          |  Webブラウザ  |
          |  (スクリプト実行) |
          +---------------+
```

## 技術的な深堀り

**HTTPヘッダーの仕組み**

HTTPヘッダーには、セキュリティ関連のヘッダーが設定できます。例えば、`Content-Security-Policy`ヘッダーは、Webブラウザが実行できるスクリプトのソースを制限します。

* `Content-Security-Policy: script-src 'self';`

**セキュリティ関連のHTTPヘッダー**

| ヘッダー名 | 説明 |
| --- | --- |
| Content-Security-Policy | スクリプトのソースを制限します。 |
| X-Frame-Options | フレームのソースを制限します。 |
| X-XSS-Protection | XSSの保護を有効にします。 |

## 攻撃・脆弱性の観点

**XSSの攻撃例**

* **Cookieの盗難**: 悪意のあるスクリプトが、ユーザーのCookieを盗みます。
* **未承認の操作**: 悪意のあるスクリプトが、ユーザーに代わって操作を実行します。

**有名なXSSの攻撃事例**

* **MySpaceのXSS攻撃** (2005年)
* **Yahoo!のXSS攻撃** (2013年)

## 対策・ベストプラクティス

**XSSの対策**

* **入力データの検証**: ユーザーが入力したデータを検証します。
* **エスケープ**: 出力データをエスケープします。
* **セキュリティ関連のHTTPヘッダー**: セキュリティ関連のHTTPヘッダーを設定します。

**XSSの防止のための設定例**

* `Content-Security-Policy: script-src 'self';`
* `X-Frame-Options: DENY`
* `X-XSS-Protection: 1; mode=block`

## 試験対策ポイント

| 試験対策ポイント | 解説 |
| --- | --- |
| XSSの種類 | 反射型XSS、ストアド型XSS、DOM型XSS |
| XSSの仕組み | 悪意のあるスクリプトの実行 |
| セキュリティ関連のHTTPヘッダー | Content-Security-Policy, X-Frame-Options, X-XSS-Protection |
